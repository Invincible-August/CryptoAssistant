## 2. 仓库结构（单仓前后端）

```
TradingAgent/
├── backend/                 # Python / FastAPI
│   ├── app/
│   │   ├── main.py          # 应用入口、生命周期、挂载 /api/v1
│   │   ├── core/            # 配置、数据库、Redis、安全、日志、异常
│   │   ├── models/          # SQLAlchemy ORM（表定义）
│   │   ├── schemas/         # Pydantic 请求/响应模型（API 契约）
│   │   ├── api/
│   │   │   ├── deps.py      # get_db、get_current_user 等依赖
│   │   │   ├── router.py    # 汇总所有 v1 子路由
│   │   │   └── v1/          # 14 组 REST 路由（与业务一一对应）
│   │   ├── services/        # 业务服务层（推荐：复杂逻辑放这里）
│   │   ├── modules/         # 评分、特征流水线、插件加载等引擎
│   │   ├── indicators/      # 技术指标插件（builtins + custom）
│   │   ├── factors/         # 量化因子插件（builtins + custom）
│   │   ├── datafeeds/       # 交易所适配器（Binance 等）
│   │   ├── backtest/        # 回测引擎、模拟成交、绩效指标
│   │   ├── lightweight_charts_compat/  # 图表格式兼容映射层（Lightweight Charts）
│   │   ├── ai/              # OpenAI 相关（模块开关控制）
│   │   ├── execution/       # 下单辅助（默认关闭）
│   │   └── tasks/           # 定时任务
│   └── requirements.txt
├── frontend/                # React + TS + Vite + Ant Design
│   └── src/
│       ├── router/          # 路由与登录保护
│       ├── layouts/         # 主布局 / 登录布局
│       ├── pages/           # 10 个业务页面
│       ├── services/        # axios 封装 + 按域划分的 API 调用
│       └── store/           # Zustand（如 auth）
├── scripts/                 # 初始化、种子数据等
└── .env / .env.example      # 环境变量（勿提交密钥）
```

## 3. 请求链路（改功能时按层下钻）

```
浏览器
  → Vite dev: /api → 代理到 localhost:8000
  → FastAPI app.main
  → /api/v1/{auth|market|...}  （见 app/api/router.py）
  → app/api/v1/*.py  （路由：参数校验、鉴权、调用 service 或引擎）
  → app/services/*.py 或 app/backtest 等  （业务与领域逻辑）
  → app/models + SQLAlchemy  （持久化）
```

**约定：**

- 对外 JSON 形状尽量用 `app/schemas` 与 `ResponseBase` 统一包装（与前端 `request` 拦截器里 `code === 200` 的判断一致）。
- 数据库会话通过 `Depends(get_db)` 注入，不要在路由里手写全局 session。

## 4. 后端：14 组 API 与代码位置

`app/api/router.py` 将前缀注册到 `/api/v1` 下，例如完整路径为 `/api/v1/market/...`。

| 前缀（tags） | 文件 | 典型职责 |
|--------------|------|----------|
| 认证 | `v1/auth.py` | 登录、JWT |
| 用户管理 | `v1/users.py` | 用户 CRUD |
| 实时监控 | `v1/monitor.py` | 监控列表、状态 |
| 行情数据 | `v1/market.py` | K 线、成交、订单簿等 |
| 技术指标 | `v1/indicators.py` | 列表、计算、结果查询 |
| 量化因子 | `v1/factors.py` | 列表、计算、结果查询 |
| 行为分析 | `v1/analysis.py` | 分析流水线入口 |
| 交易建议 | `v1/signals.py` | 信号与建议 |
| 回测 | `v1/backtest.py` | 回测任务与结果 |
| 执行辅助 | `v1/execution.py` | 下单相关（模块开关） |
| AI 分析 | `v1/ai.py` | OpenAI（模块开关） |
| 图表分析 | `v1/chart.py` | `GET /chart/bundle`：K 线 + 可选指标 JSON（Lightweight Charts），不依赖 TradingView webhook/兼容接口 |
| 系统配置 | `v1/config.py` | 模块开关、系统配置 |
| 系统日志 | `v1/logs.py` | 日志查询 |

**服务层现状（`app/services/`）：**  
当前包含 `auth`、`user`、`market`、`indicator`、`factor`、`analysis`、`module`、`log` 等。若某路由逻辑很长，优先抽到 `services` 或独立 `xxx_engine`，保持 `v1` 文件薄。

## 5. 前端：路由与 API 封装

### 5.1 页面路由（`frontend/src/router/index.tsx`）

| 路径 | 页面组件 | 说明 |
|------|----------|------|
| `/login` | `Login` | 登录 |
| `/` | `Dashboard` | 仪表盘 |
| `/monitor` | `Monitor` | 实时监控 |
| `/chart` | `ChartAnalysis` | 图表分析 |
| `/indicators` | `Indicators` | 技术指标 |
| `/factors` | `Factors` | 量化因子 |
| `/backtest` | `Backtest` | 策略回测 |
| `/ai` | `AIAnalysis` | AI 分析 |
| `/settings` | `Settings` | 设置 |
| `/logs` | `Logs` | 日志 |

侧栏菜单在 `layouts/BasicLayout.tsx` 的 `menuItems`，新增页面时需同步路由与菜单。

### 5.2 HTTP 客户端（`frontend/src/services/request.ts`）

- `baseURL` 为 `/api/v1`（开发时由 Vite 代理到后端）。
- 自动带 `Authorization: Bearer <token>`。
- 约定响应体含 `code`，非 200 会 `message.error` 并 reject。

按业务拆分的封装在 `services/*.ts`（如 `market.ts`、`backtest.ts`）。**新增页面时建议新建或扩展对应 `services` 文件**，避免在组件里到处写裸 `request.get` 路径字符串。

## 6. 功能完成度与二次开发侧重点（诚实说明）

MVP 中部分页面仍为**占位或简化实现**，二次开发时可优先对接真实数据与交互：

| 区域 | 现状提示 | 建议改动入口 |
|------|----------|--------------|
| **图表分析** (`pages/ChartAnalysis`) | 已接 `GET /api/v1/chart/bundle` + `lightweight-charts`；需库中有 K 线 | 后端 `services/chart_data_service.py`、`chart_series_builder.py`；前端 `components/chart/LightweightChartBoard.tsx`、`services/chart.ts` |
| **回测** | 后端 `backtest/` 引擎与 API 相对完整；前端需与接口字段对齐 | `backend/app/backtest/engine.py`、`v1/backtest.py`；前端 `services/backtest.ts` |
| **实时监控** | 列表类接口已接；深度实时推送、WS 需按产品补全 | `datafeeds/`、`tasks/`、以及 monitor 相关 API |
| **AI / 执行** | 受 `.env` 与模块开关控制，未启用时接口可能拒绝或降级 | `settings` 与 `MODULE_*_ENABLED` |

开发前在 **Swagger** 中试调目标接口，确认返回结构再写前端类型与表格/图表。

**图表分析模块**的逐文件、逐函数与数据流说明（含模仿开发步骤）见 **[图表分析开发指南.md](./图表分析开发指南.md)**。

## 7. 常见扩展任务（按步骤做）

### 7.1 新增一个 REST 接口

1. 在 `app/schemas/` 增加或复用请求/响应模型。  
2. 在 `app/api/v1/` 对应文件中加 `@router.get/post/...`。  
3. 复杂逻辑放到 `app/services/` 或领域模块。  
4. 需要落库时改 `app/models/` 并执行迁移策略（当前开发环境可用 `init_db` 建表；生产应用 Alembic）。  
5. 打开 `/docs` 验证；前端在 `services/` 增加方法并在页面调用。

### 7.2 新增一个前端页面

1. `pages/YourFeature/index.tsx` 写页面。  
2. `router/index.tsx` 增加 `<Route>`。  
3. `BasicLayout.tsx` 增加菜单项。  
4. `services/yourFeature.ts` 封装 API。

### 7.3 新增技术指标

1. 在 `app/indicators/builtins/` 或 `custom/` 新建文件，继承 `BaseIndicator`，实现 `calculate()`，设置 `indicator_key` / `name` 等。  
2. 启动时 `register_all_indicators()` 会自动扫描注册（见 `app/indicators/__init__.py`）。  
3. README 中有自定义指标示例可参考。

### 7.4 新增量化因子

1. 在 `app/factors/builtins/` 或 `custom/` 继承 `BaseFactor`，实现计算逻辑，设置 `factor_key`。  
2. `register_all_factors()` 会扫描注册。  
3. 若需新数据源，检查 `factor_service` 与 `datafeeds` 是否提供上下文。

### 7.5 接入交易所或统一行情

1. 实现 `app/datafeeds/base.py` 中的 `BaseExchangeAdapter` 子类。  
2. 在 `DatafeedManager` 中注册。  
3. 将统一结构写入 DB 或交给现有 `market_service`。

## 8. 配置与模块开关

环境变量模板：`.env.example`。关键项：

- `DATABASE_URL`、`REDIS_URL`  
- `MODULE_AI_ENABLED`、`MODULE_EXECUTION_ENABLED`、`MODULE_BACKTEST_ENABLED`  
- `BINANCE_*`、`OPENAI_*`（勿提交真实密钥）

业务代码中通过 `app.core.config.settings` 读取；部分能力在运行时还会结合 DB 中的模块配置（见 `module_service` 与 CHANGELOG 说明）。

## 10. 推荐阅读顺序（新成员）

1. 本文档 + `项目大纲.md`  
2. `app/main.py` → `app/api/router.py` → 挑一个你最关心的 `v1/*.py` 读到底  
3. 对应 `services` 与 `models`
