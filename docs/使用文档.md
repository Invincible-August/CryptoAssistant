# 使用文档

## 环境准备

### 必需软件

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| Python | 3.12+ | 后端运行环境 |
| Node.js | 20+ | 前端构建和开发 |
| PostgreSQL | 16+ | 主数据库 |
| Redis | 7+ | 缓存和实时数据（Windows可用Memurai替代） |

### Windows 安装说明

1. **Python**: 从 https://www.python.org/downloads/ 下载安装，勾选"Add to PATH"
2. **Node.js**: 从 https://nodejs.org/ 下载LTS版本安装
3. **PostgreSQL**: 从 https://www.postgresql.org/download/windows/ 下载安装，安装时记住设置的密码
4. **Redis**: Windows没有官方Redis，可选择以下方案：
   - 安装 Memurai: https://www.memurai.com/
   - 使用 WSL2 中的 Redis: `wsl sudo apt install redis-server && wsl redis-server`
   - 系统也支持 Redis 不可用时降级运行（不使用缓存）

### Linux / macOS 安装说明

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install python3.12 python3.12-venv postgresql redis-server nodejs npm

# macOS (Homebrew)
brew install python@3.12 postgresql@16 redis node
```

## 快速启动

### 第一步：配置环境变量

```bash
cd TradingAgent
copy .env.example .env          # Windows
# cp .env.example .env          # Linux/macOS
```

编辑 `.env` 文件，至少确认以下配置正确：

```ini
DATABASE_URL=postgresql+asyncpg://postgres:你的密码@localhost:5432/crypto_assistant
REDIS_URL=redis://localhost:6379/0
```

### 第二步：创建数据库

连接 PostgreSQL 并创建数据库：

```bash
# 使用psql命令行
psql -U postgres
```

```sql
CREATE DATABASE crypto_assistant;
\q
```

或者使用 pgAdmin 图形界面创建。

### 第三步：启动 Redis

```bash
redis-server
```

如果没有安装 Redis，系统也可以启动（会有警告，但不影响核心功能）。

### 第四步：安装后端依赖并启动

```bash
cd backend
pip install -r requirements.txt

# 回到项目根目录
cd ..

# 创建管理员账户
python scripts/create_admin.py

# 导入示例数据（含30天BTCUSDT的1小时K线）
python scripts/seed_demo_data.py

# 启动后端
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

启动成功后可以看到：
```
INFO     | 系统启动完成 - 访问 http://0.0.0.0:8000/docs 查看API文档
```

### 第五步：安装前端依赖并启动

**打开一个新的终端窗口**：

```bash
cd frontend
npm install
npm run dev
```

启动成功后可以看到：
```
  VITE v6.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
```

### 第六步：登录系统

1. 浏览器访问 http://localhost:5173
2. 输入用户名: `admin`
3. 输入密码: `admin123456`
4. 进入仪表盘

## 功能使用说明

### 仪表盘
- 展示系统概览：监控交易对数、注册指标数、回测任务数、AI分析次数
- 展示系统状态：后端/数据库/Redis/各模块的运行状态
- 快速导航到各功能页面

### 实时监控
1. 进入「实时监控」页面
2. 查看当前监控的交易对列表（种子数据包含BTCUSDT和ETHUSDT）
3. 点击「添加监控」可以添加新的交易对
4. 监控状态：active=监控中，paused=暂停，stopped=停止

### 技术指标
1. 进入「技术指标」页面查看所有已注册的指标
2. 来源标签区分：蓝色=系统内置，绿色=人工自定义，紫色=AI生成
3. 选择指标点击「计算」即可对指定交易对计算结果
4. 自定义指标：在 `backend/app/indicators/custom/` 目录下创建Python文件，继承 `BaseIndicator`

### 量化因子
1. 进入「量化因子」页面查看所有已注册的因子
2. 因子可以依赖多种数据源（K线、成交、深度、OI等）
3. 因子评分统一在0-100范围内，50为中性
4. 自定义因子：在 `backend/app/factors/custom/` 目录下创建Python文件，继承 `BaseFactor`

### 图表分析
1. 进入「图表分析」页面
2. 选择交易对和K线周期
3. 查看K线图和指标覆盖层（需要有K线数据）

### 策略回测
1. 进入「策略回测」页面
2. 填写回测参数：
   - 任务名称
   - 交易对（默认BTCUSDT）
   - K线周期（默认1小时）
   - 时间范围
   - 初始资金（默认10000 USDT）
3. 点击「开始回测」
4. 查看绩效指标：总收益率、最大回撤、胜率、盈亏比、夏普比率
5. 注意：需要先通过seed_demo_data.py导入历史K线数据

### AI分析
1. 先在「系统设置」中启用AI模块
2. 在 `.env` 中配置 `OPENAI_API_KEY`
3. 进入「AI分析」页面，选择交易对
4. 点击「运行AI分析」
5. AI生成的所有内容都会标记 `source=ai`
6. 可对AI结果提交反馈

### 系统设置
- 管理模块开关（AI、TradingView、执行辅助、回测、交易所）
- 仅管理员角色可以访问
- 开关变更立即生效

### 系统日志
- 查看系统运行日志（按级别和模块过滤）
- 查看错误日志（按模块过滤）

## 模块开关说明

| 模块 | 环境变量 | 默认状态 | 说明 |
|------|---------|---------|------|
| AI | MODULE_AI_ENABLED | 关闭 | 需配置OPENAI_API_KEY |
| TradingView | MODULE_TRADINGVIEW_ENABLED | 关闭 | 需要TradingView警报配置 |
| 执行辅助 | MODULE_EXECUTION_ENABLED | 关闭 | 默认模拟模式，不操作真实资金 |
| 回测 | MODULE_BACKTEST_ENABLED | 开启 | 需要历史K线数据 |

环境变量优先级高于数据库配置。

## 生产部署（可选）

开发阶段使用 Vite 开发服务器 + uvicorn 即可。如果需要生产部署：

### 方案：Nginx 前后端分离

1. **构建前端**：
```bash
cd frontend
npm run build
```

2. **安装 Nginx** 并将 `deploy/nginx.conf` 复制到 Nginx 配置目录

3. **修改 nginx.conf**：
   - `root` 指向 `frontend/dist` 的绝对路径
   - `proxy_pass` 指向后端地址 `http://127.0.0.1:8000`

4. **后端多进程运行**：
```bash
cd backend
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

5. 重启 Nginx

## 常见问题

### Redis连接失败
系统启动时如果Redis不可用，会输出警告但不会崩溃。核心功能正常，只是不使用缓存。

### 数据库连接失败
确认PostgreSQL已启动，且 `.env` 中的 `DATABASE_URL` 配置正确（特别是密码和端口）。

### 回测无数据
需要先运行 `python scripts/seed_demo_data.py` 导入示例K线数据。

### Windows下Redis替代方案
- 安装 Memurai（Redis for Windows兼容实现）
- 或在 WSL2 中安装并运行Redis
- 或直接跳过Redis，系统会降级运行
